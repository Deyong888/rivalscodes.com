## 1. 产品概述

### 1.1 产品定位
将 rivalscodes.com 从单一的代码分发平台升级为 Roblox Rivals 玩家的全方位游戏工具平台，通过AI驱动的实用工具提升用户留存率和平台价值。

### 1.2 产品愿景
成为 Roblox Rivals 玩家日常游戏生活中不可或缺的"工具伴侣"，从"每月访问一次"的站点转变为"每次打开游戏前都会查看"的战术终端。

### 1.3 核心价值主张
- **解决信息不对称**：通过AI技术自动化处理繁杂的游戏数据
- **降低学习成本**：提供直观的可视化工具替代复杂的计算
- **实时数据更新**：动态反映游戏版本变化，保持内容时效性
- **社区生态建设**：打造玩家分享和交流的平台

## 2. 目标用户

### 2.1 主要用户画像

#### 用户群体A：硬核竞技玩家（核心用户）
- **特征**：追求极致竞技表现，对游戏数值极其敏感
- **需求**：灵敏度转换、武器数据分析、Meta追踪
- **行为**：频繁搜索"最佳设置"、"伤害数据"、"排名系统机制"
- **价值**：高留存率、高参与度、愿意分享工具

#### 用户群体B：内容创作者
- **特征**：YouTuber、Twitch主播、社区意见领袖
- **需求**：准星代码分享、武器数据可视化、Meta分析
- **行为**：需要专业工具支撑内容创作
- **价值**：带来流量、提升平台权威性

#### 用户群体C：收藏与交易玩家
- **特征**：关注虚拟资产价值、参与社区交易
- **需求**：资产估值、价格追踪、库存计算
- **行为**：频繁查询物品价值、分享库存价值
- **价值**：病毒式传播、社交分享

### 2.2 用户痛点总结
1. **灵敏度设置混乱**：不理解Rivals的乘数机制，无法准确转换其他游戏的灵敏度
2. **准星代码传播低效**：需要手动输入长串代码，无法预览效果
3. **武器数据滞后**：静态攻略过时快，无法反映版本变化
4. **资产价值不透明**：依赖分散的表格和视频，缺乏统一的价值参考

## 3. 功能需求

### 3.1 工具一：高精度灵敏度转换与校准套件
**工具名称**: Rivals True-Aim Calibrator（Rivals 真·瞄准校准器）

#### 3.1.1 功能列表

**F1.1 跨游戏精准转换（Bi-Directional Conversion）**
- **输入参数**：
  - 源游戏（Valorant / CS:GO / CS2 / Apex Legends / 其他Roblox射击游戏）
  - 源灵敏度值
  - 鼠标DPI
  - 目标：Rivals
- **输出结果**：
  - cm/360（每转360度所需的鼠标移动距离）
  - 推荐的Roblox基础灵敏度
  - 推荐的Rivals游戏内滑块数值
  - 转换公式说明
- **反向转换**：
  - 支持从Rivals转换到其他游戏
  - 保持相同的cm/360值

**F1.2 乘数效应可视化（Multiplier Visualizer）**
- **功能描述**：图表展示Roblox基础设置与游戏内滑块的相互作用
- **可视化内容**：
  - 交互式滑块：实时调整Roblox灵敏度和Rivals倍率
  - 实时计算显示：最终eDPI、cm/360
  - 对比图表：不同组合的效果对比
- **教育性内容**：解释为什么"0.2 Roblox Sens × 2.0 Rivals Multiplier"等同于"0.4 Roblox Sens × 1.0 Rivals Multiplier"

**F1.3 AI手感诊断师（AI Aim Doctor）**
- **输入方式**：自然语言描述问题
  - 示例："我在近距离对枪时总是跟不上目标的移动，但在远距离又觉得太滑"
- **AI分析**：
  - 识别问题类型（近距离/远距离/微调/跟枪）
  - 分析游戏机制（角速度、像素跳跃）
  - 生成个性化调整方案
- **输出建议**：
  - DPI调整建议
  - 游戏内倍率调整建议
  - 解释调整原理

**F1.4 灵敏度预设库**
- **功能**：保存常用配置
- **预设来源**：
  - 知名玩家配置（Nosniy、TanqR、Kaye等）
  - 按游戏分类的推荐配置
- **功能**：一键应用预设

#### 3.1.2 技术实现要求
- **前端**：React组件，使用Canvas或SVG进行可视化
- **计算逻辑**：JavaScript实现灵敏度转换算法
- **AI集成**：集成OpenAI API或Claude API进行自然语言处理
- **数据存储**：本地存储（localStorage）保存用户配置

#### 3.1.3 SEO关键词策略
- 核心关键词：`Roblox Rivals sensitivity converter`, `Rivals aim settings calculator`, `Valorant to Rivals sensitivity`
- 长尾关键词：`How to match Rivals sens with Roblox`, `Rivals sensitivity multiplier explained`

### 3.2 工具二：可视化准星代码库与生成器
**工具名称**: Rivals Crosshair Architect（Rivals 准星架构师）

#### 3.2.1 功能列表

**F2.1 可视化代码库（Visual Repository）**
- **准星数据库**：
  - 收录知名玩家准星（Nosniy、TanqR、Kaye等）
  - 按类型分类：点状（Dot）、十字（Cross）、圆环（Circle）
  - 按用途分类：狙击、近战、通用
- **预览功能**：
  - 实时预览准星效果
  - 支持不同背景（亮色/暗色/游戏地图背景）
  - 对比度测试工具
- **元数据**：
  - 准星名称、来源、使用次数、创建日期
  - 标签系统：颜色、大小、动态效果

**F2.2 一键复制与解析（One-Click Import）**
- **复制功能**：
  - 一键复制符合游戏格式的代码
  - 支持多种格式输出
- **代码解析**：
  - 用户粘贴代码后自动解析
  - 可视化显示参数：
    - 粗细（Thickness）
    - 间距（Gap）
    - 颜色（RGB值）
    - 透明度
    - 动态效果参数
- **参数编辑**：
  - 可视化编辑器调整参数
  - 实时预览效果
  - 生成新代码

**F2.3 AI图像转代码（AI Image-to-Code）**
- **功能描述**：上传准星截图，AI自动识别并生成代码
- **技术实现**：
  - 计算机视觉识别准星形状
  - 识别像素宽度、颜色值、中心点
  - 逆向生成Rivals代码字符串
- **使用场景**：
  - 从视频/直播中截图获取准星
  - 自定义准星设计转换为代码

**F2.4 准星编辑器**
- **可视化编辑界面**：
  - 拖拽调整位置
  - 滑块调整参数
  - 颜色选择器
  - 实时预览
- **高级功能**：
  - 动态效果设置（开火扩散、移动扩散）
  - 预设模板
  - 导出/导入配置

#### 3.2.2 数据结构设计

**准星数据模型**：
```typescript

    thickness: number;

    opacity: number;
    // ... 其他参数
  };
}
```

### 3.3 工具三：动态武器数据与Meta分析引擎
**工具名称**: Rivals Meta Lab（Rivals 版本实验室）

#### 3.3.1 功能列表

**F3.1 交互式数据卡片（Interactive Stat Cards）**
- **武器档案**：为每把武器建立详细数据档案
  - 基础伤害（Body Damage）
  - 爆头倍率（Crit Multiplier）
  - 射速（Fire Rate，秒/发）
  - DPS（每秒伤害）
  - TTK（Time to Kill，击杀所需时间）
  - 移动速度惩罚/加成
  - 特殊机制描述（如武士刀格挡、火焰喷射器持续伤害）
- **可视化展示**：
  - 雷达图对比多把武器
  - 伤害衰减曲线图
  - 射程效果图

**F3.2 TTK模拟器（Time-to-Kill Calculator）**
- **输入参数**：
  - 目标血量（100/150/200等）
  - 防御状态（无护盾/有护盾/超护盾）
  - 命中部位（身体/头部）
  - 命中率（可选）
- **计算输出**：
  - 理论击杀时间
  - 所需命中次数
  - 伤害分布图
- **对比功能**：
  - 多武器对比
  - 不同场景对比（近距离/中距离/远距离）

**F3.3 AI驱动的Meta简报（AI Meta Summary）**
- **数据源监控**：
  - 官方Discord更新日志
  - Reddit讨论热度
  - YouTube视频提及频率
- **自动生成内容**：
  - 每周Meta变动报告
  - 武器使用率趋势
  - 版本答案分析
  - 示例："本周RPG讨论热度下降20%，因为更新将溅射伤害锁定在50点。相反，由于移动速度加成，匕首在高端局使用率攀升。"
- **更新提醒**：重大平衡调整时自动推送

**F3.4 武器梯队排行榜（Tier List）**
- **动态排名**：
  - 基于数据计算的客观排名
  - 社区投票的主观排名
  - 综合排名
- **分类展示**：
  - 按武器类型（主武器/副武器/近战）
  - 按使用场景（近距离/中距离/远距离）
  - 按游戏模式（排位/休闲）

#### 3.3.2 武器数据模型

**武器属性矩阵**：
```typescript

  id: string;
```

## 4. 非功能性需求

### 4.1 性能要求
- **页面加载时间**：首屏加载时间 < 2秒
- **工具响应时间**：计算类工具响应时间 < 500ms
- **AI处理时间**：AI功能响应时间 < 5秒
- **并发支持**：支持至少1000并发用户
- **数据更新频率**：武器数据每日更新，资产价格每小时更新

### 4.2 可用性要求
- **浏览器兼容性**：支持Chrome、Firefox、Safari、Edge最新版本
- **移动端适配**：响应式设计，支持移动端访问
- **无障碍性**：符合WCAG 2.1 AA标准
- **多语言支持**：支持中文、英文（后续扩展其他语言）

### 4.3 安全性要求
- **数据安全**：用户输入数据不存储敏感信息
- **API安全**：AI API密钥服务器端管理，不暴露给前端
- **XSS防护**：所有用户输入进行转义处理
- **CSRF防护**：实施CSRF令牌验证
- **速率限制**：API调用实施速率限制，防止滥用

### 4.4 可维护性要求
- **代码规范**：遵循项目现有代码规范（ESLint配置）
- **文档完善**：关键功能提供代码注释和文档
- **模块化设计**：工具功能模块化，便于独立开发和维护
- **版本控制**：使用Git进行版本管理，遵循语义化版本

### 5.3 数据流设计

**灵敏度转换工具数据流**：
```
用户输入 → 前端验证 → 本地计算 → 显示结果
         ↓ (需要AI诊断)
       API调用 → OpenAI API → 返回建议 → 显示结果
```

**准星工具数据流**：
```
用户上传图片 → API调用 → OpenAI Vision → 解析参数 → 生成代码 → 返回前端
用户输入代码 → 前端解析 → 渲染预览 → 显示参数
```

**武器数据工具数据流**：
```
用户查询 → 读取JSON数据 → 前端计算DPS/TTK → 显示结果
数据更新 → 爬虫服务 → 更新JSON文件 → GitHub提交 → 触发重新部署
```

**资产估值工具数据流**：
```
用户查询 → 读取JSON数据 → 显示价格信息
价格更新 → 爬虫服务 → 抓取Discord/表格 → 计算加权平均 → 更新JSON
```

## 7. API设计

### 7.1 AI相关API

**POST /api/ai/aim-diagnosis**
- **功能**：AI手感诊断
- **请求体**：
```typescript
{
  description: string;  // 用户描述的问题
  currentSettings?: {   // 当前设置（可选）
    dpi: number;
    robloxSens: number;
    rivalsMultiplier: number;
  };
}
```
- **响应**：
```typescript
{
  suggestions: {
    dpiAdjustment?: number;
    robloxSensAdjustment?: number;
    rivalsMultiplierAdjustment?: number;
    explanation: string;  // 调整原理说明
  };
}
```

**POST /api/ai/image-to-crosshair**
- **功能**：图像转准星代码
- **请求体**：FormData（包含图片文件）
- **响应**：
```typescript
{
  code: string;  // 生成的准星代码
  parameters: {
    thickness: number;
    gap: number;
    color: [number, number, number];
    // ... 其他参数
  };
  confidence: number;  // 识别置信度
}
```

**POST /api/ai/meta-summary**
- **功能**：生成Meta简报
- **请求体**：
```typescript
{
  timeframe: 'week' | 'month';  // 时间范围
}
```
- **响应**：
```typescript
{
  summary: string;  // AI生成的简报文本
  weaponTrends: {
    weaponId: string;
    trend: 'up' | 'down' | 'stable';
    changePercent: number;
  }[];
  lastUpdated: string;
}
```

### 7.2 数据查询API

**GET /api/weapons**
- **功能**：获取武器列表
- **查询参数**：
  - `slot`: 'Primary' | 'Secondary' | 'Melee'
  - `sortBy`: 'dps' | 'ttk' | 'name'
- **响应**：Weapon[]数组

**GET /api/weapons/[id]**
- **功能**：获取单个武器详情
- **响应**：Weapon对象

**GET /api/weapons/[id]/ttk**
- **功能**：计算TTK
- **查询参数**：
  - `health`: number
  - `armor`: boolean
  - `headshot`: boolean
- **响应**：
```typescript
{
  ttk: number;  // 秒
  shotsRequired: number;
  damageBreakdown: number[];
}
```

**GET /api/assets**
- **功能**：获取资产列表
- **查询参数**：
  - `type`: 'Wrap' | 'Charm' | 'Finisher'
  - `search`: string
- **响应**：Asset[]数组

**GET /api/assets/[id]**
- **功能**：获取单个资产详情
- **响应**：Asset对象

**GET /api/assets/[id]/history**
- **功能**：获取价格历史
- **查询参数**：
  - `period`: 'week' | 'month' | '3months' | 'year'
- **响应**：
```typescript
{
  history: {
    date: string;
    value: number;
  }[];
}
```

**GET /api/crosshairs**
- **功能**：获取准星列表
- **查询参数**：
  - `type`: 'Dot' | 'Cross' | 'Circle'
  - `tags`: string[]  // 标签筛选
  - `sortBy`: 'popular' | 'newest' | 'name'
- **响应**：Crosshair[]数组

### 8.2 数据更新机制

**武器数据更新**：
- 手动更新：通过管理面板更新JSON文件
- 自动更新：爬虫服务定期检查游戏更新，提交PR到GitHub
- 触发部署：GitHub Webhook触发Vercel重新部署

**资产价格更新**：
- 定时任务：每小时运行一次爬虫

- **静态数据**：武器、准星数据使用Next.js静态生成，ISR（增量静态再生）
- **动态数据**：资产价格使用SWR客户端缓存，5分钟刷新
- **AI结果**：不缓存，每次实时计算
